# Wyckoff Strategy Playbook (量化架构视角)

本文档描述策略框架与量化执行流程，整体架构采用“日线级 + 结构化强风控 + 人机协同”的设计理念：
通过漏斗广域初筛、TopN 量化打分兜底、大模型语义盘口决策，以及 OMS 端的严格风控，最终输出确定下的交易建议。
（注：实际的滑点、风险占比、极值窗口等阈值均不在本档公开披露，具体以系统 `.env` 配置为准。）

覆盖模块：
- Step2 漏斗选股：`scripts/wyckoff_funnel.py` + `core/wyckoff_engine.py` (L1-L4 数学形态初筛与打分)
- Step3 批量 AI 研报：`scripts/step3_batch_report.py` (LLM 两极分类战报构建)
- Step4 私人再平衡 OMS：`scripts/step4_rebalancer.py` (持仓管理与实盘风控阻断)

## 1. 系统调度与数据工程

### 1.1 定时与环境对齐
- 由 GitHub Actions 在工作日自动化调度（同分支并发互斥机制避免任务重叠）。
- 统一切换目标交易日（T / T-1）：由 `utils/trading_clock.py` 判定，严格依据盘后及盘中时间节点切分。
- **快照补齐与对齐检查：** 若三方日线数据未落库，可依靠本地 Snapshot 补齐最后一根 K 线进行计算。

### 1.2 Snapshot 快照与本地回测器
- 每轮日线全量拉取后，会将 4000+ 股票行情快照序列化到本地 `data/funnel_snapshots` 目录。
- **意义：** 从根本上分离了“网络取数”与“量化算子”，系统支持秒级重播（Replay），大幅提升了因子调优、正则参数改进以及回测的工程效率。
- **当前默认回测口径（2026-03）**：`hold_days=5`、`exit_mode=sltp`、`stop_loss=-9%`、`take_profit=0`（只做止损，不预设止盈），并在 summary 中强制输出 `最大回撤 / VaR95 / CVaR95 / 最长连亏` 四项风险指标。

### 1.3 盘前风控预警 (Premarket Risk)
- **大盘情绪温度计**：在每日实际开盘前（08:30），独立调度风控任务（`scripts/premarket_risk_job.py`）。
- **监控标的**：富时 A50 期指（反映 A 股核心资产开盘意愿）与 VIX 恐慌指数（多级降级保障，反映全球高风险避险情绪）。
- **物理熔断前哨**：通过预判 `RISK_OFF` 或 `BLACK_SWAN`（黑天鹅）状态，向系统及交易者提前告警，从宏观面有效避免在情绪崩溃日盲目开仓。

## 2. Step2：Wyckoff Funnel（四层多维漏斗）

### 2.1 Step0：大盘宏观总闸
- **宏观水温：** 通过指数收盘价与 MA50/MA200 交叉、均线短期斜率及单边涨跌幅，归类市场环境（`RISK_ON` / `NEUTRAL` / `RISK_OFF` / `CRASH`）。
- **🌟 强力悬崖检测 (Crash 极寒模式)：** 同步监测主板与小盘股单日暴跌幅度（如 -1.5% 和 -3%），以及市场广度单日断崖式下降（赚了指数不赚钱的虚假诱多）。一旦触发，系统直接切入 `CRASH` 冰封状态。
- **动态自适应调参：** 冷水温时全局提升漏斗参数；遇到 `CRASH` 时漏斗门槛提升至极限（过滤掉 99% 的跟跌股）；热水温适度放开强度过滤区间。

### 2.2 漏斗阶梯与量化因子
#### Layer1：流动性剔除
- 过滤非主板/非创业板样本。
- 采用移动平均平滑，刚性切除流动性干涸、微盘、ST 风险股。

#### Layer2：截面强弱甄别（顺势）
- 结构条件：必须处于大多头排列（或大盘恐慌连跌时抗跌守护 MA20）。
- **动量拦截 (RS/RPS)：** 基于两段不同周期的欧奈尔相对强度（RPS）实施 75%-85% 的截面淘汰，只在市场顶部选手里寻猎。

#### Layer3：板块共振与 Top15 Alpha Score 打分
- **共振识别：** 不僵化套用固定前 N 个行业，而是根据“行业通过率（L2分布相对于L1基础分布）+ 行业强度截面中位数”构建二重动态过滤。
- **🌟 兜底排名打分机制（Rank L3）：** 如果在 L4 未捕捉到极度完美的微观形态，则对整体 L3 幸存者做复合因子打分（动量+干涸度+板块加成），强制推送全市场最强共振且技术到位的 Top15 股票（这有效对抗了极度严苛正则导致的空池问题）。

#### Layer4：微观形态捕捉（威科夫扳机）
基于实战演进（容许微小刺穿、真实滑点），对个股执行日线切片捕捉：
- **Spring（终极震仓）：** 下探支撑区并强力收回、放巨量。
- **LPS（缩量回踩）：** 特定均线支撑位的极值缩量表现（加入容差率，避免主力的悬空底或轻微刺穿底错过）。
- **EVR (Effort vs Result)：** 高位派发保护下，捕获“放量而不跌”的核心换手背离信号。

## 3. Step3：AI "Tape Reader" (盘口翻译大模型)

抛弃了传统的“把生数据全塞给 AI 试图让其算数”思维，完全依靠 Step2 切片数据驱动 LLM。

### 3.1 高光特征注入（Feature Slice）
- 系统并不把涵盖一切的 OHLC 文本扔给 AI，而是压缩给出**结构背景（均价乖离率验证位阶）**和 **近15日量价切片（缩量确认日）**。
- **核心足迹捕捉：** 专门提炼出过去 60 天内极度缩容、暴涨暴跌或巨量分歧的高光切片日期，供 AI 像作弊一样直接推演主力行为。

### 3.2 极简二分类决策（No More Over-Engineering）
- 剥离人工设定的复杂压缩策略。
- **硬约束 Prompt：** 强制 AI 在所有传入标的中，以威科夫 Composite Man 的视角，做且仅做二维截断输出：“👀 继续观察” 和 “🚀 立刻建仓”。
- **强制给锚：** 买卖口径不得自创孤立数字，必须依据传入的“最新实际收盘价”框出战区、给出量价确认条件及结构止损。

### 3.3 RAG 舆情防雷
- 结合外部检索引擎实施最后一公里负面信息扫清，一票否决违规、立案等高风险舆情标的。

## 4. Step4：私人再平衡 OMS (订单管理系统)

OMS 层是保证这套系统不出致命灾难的“最终刹车片”。

### 4.1 独立决策权与系统性物理熔断
- **边界划分：** LLM 输出的文字一律视为“建议”，真正挂单的执行权被全权接管在 Python OMS 里。
- **🌟 极寒熔断阻断器：** 当宏观检测判定为 `CRASH` 模式（或未来接入 VIX/A50 等外部预警时），OMS 将强行没收 AI 的买入权，直接物理拦截并丢弃所有 `PROBE/ATTACK` 开仓指令，仅允许持有、止盈或止损。
- **优先级防线：** `EXIT（强制止损） > TRIM（止盈） > HOLD > PROBE / ATTACK（建仓）`。

### 4.2 资产与交易纪律
- 依靠 Supabase / 云端环境记录持仓账本。
- 根据总账户权益（NAV）分配硬性预算占比和单次最大资金风险上限。
- 内置 A 股按 100 股取整的刚性要求。剩余资金跑不满 100 股将直接拒单抛出 `NO_TRADE`。

### 4.3 兜底级止损与执行滑点
- **动态 ATR 跟踪防线：** 利用真实波动幅度的收缩扩张（ATR），对当前所有持仓构建跟随动态平仓线，一旦收盘击穿强制斩仓。
- **开盘滑点溢价保护：** 结合静态滑点与 ATR 值算出可接受的买入滑点容忍带，有效避开集合竞价被无脑高开骗炮。
- **买入硬止损（当前默认）：** 对 `PROBE/ATTACK` 新开仓动作，OMS 使用 `floor` 模式兜底 `-9%`（`STEP4_BUY_HARD_STOP_PCT=9`, `STEP4_BUY_STOP_MODE=floor`），只会上调过宽止损，绝不下调原本更紧的止损。
- **持仓周期硬规则：** OMS 对已有持仓执行 `max_hold_days=5` 强制清仓，到期会直接覆盖模型建议为 `EXIT`。
- **单票仓位上限：** 默认 `PROBE=10%`、`ATTACK=20%`（均按总权益上限控制），满足“Top3 分散，不单票重仓”约束。

## 5. 核心流与哲学

1. **先选容错环境，再选股票：** 用大盘广度和均线把风险期一刀切屏蔽。
2. **顺大势，逆小势：** L2 保证标的绝对处于主升浪大区间，L4 则专找大区间内缩量回档并未破结构的小级别买点。
3. **接受实战略有误差，但牢抓高确定性：** L4 放开了一点微观回测容差，配合大模型高光切片的语义辅助，极大提升系统的“人味儿”。
4. **防御体系的终极底线：** 宁可错过，绝不硬抗。基于大盘广度悬崖和 OMS 级订单冻结的双重机制，把躲避非理性黑天鹅（Bull Trap / 系统性崩盘）做成了被动技能。
5. **离线高频引擎：** Dataset Snapshot 打通了本地参数魔调，这才是从脚踏车跃进到工业风洞级别的核心。

量化设计的至高追求，不是从 5000 只股票里精确点出涨停板；而是**“由系统机器代劳，冷冰冰地隔绝 4995 只高危渣滓；把幸存的 5 份附带详尽防守带的盘口档案，呈现于你的桌面之上”**。
