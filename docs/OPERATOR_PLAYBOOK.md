# A 股操作手册（日漏斗 × 尾盘）

> 本文是**实盘怎么用**的单一口径。策略细节见 [`README_STRATEGY.md`](../README_STRATEGY.md)，执行链路见 [`A_SHARE_FUNNEL_FLOW.md`](A_SHARE_FUNNEL_FLOW.md)。

---

## 1. 一句话原则

**日漏斗定候选与环境；尾盘定今天能不能买。**  
两者串联，不是二选一。

```text
日漏斗（盘后）→ 主线候选名单
  → Step3 起跳板（建议）
  → confirmed 二次确认
  → 尾盘 BUY
  → 才下单
```

---

## 2. 双书结构

| 书 | 做什么 | 配额/仓位 | 持有 |
|----|--------|-----------|------|
| **主线趋势书** | 主题连续 + 高 RPS + 回踩 MA5/MA10/MA20 或平台再突破 | 主仓 70–80% | 约 15 日；破 MA20 / 主题缩量阴跌再减 |
| **结构观察书** | Spring / LPS / Compression 等经典结构 | 轻仓或观察 | **默认 5 日**时间止盈 |

NEUTRAL 默认 AI 配额：**Trend 5 / Accum 1**。  
RISK_ON 研究配额：**5 / 1**；只用于 AI/shadow 对照，市场闸门仍禁止正式推荐和新开仓。

---

## 3. 市场闸门（先看报告顶部）

| 水温 / 模式 | 新开仓 | 你怎么做 |
|-------------|--------|----------|
| **NEUTRAL**（`mainline_active`） | 允许 | 只做主线确认链路 |
| **CAUTION** | 观察买入 | 必须二次确认，不追 |
| **RISK_ON** | **禁止** | 只管理旧仓 |
| **BEAR_REBOUND / PANIC_REPAIR** | 禁止自动开仓 | 观察修复质量 |
| **RISK_OFF / CRASH / BLACK_SWAN** | **禁止** | 现金/减仓优先 |

报告顶部固定有 **「🧭 今日执行纪律」**，先读纪律再读候选。

---

## 4. 每日流程

### 盘后：日漏斗报告

1. 读 **执行纪律** + **今日交易模式**（禁止新仓则明日不新开）  
2. 只记 **主线买点候选**（0–3 只）；旁路/Accum 不当主仓  
3. 等 Step3 **起跳板**（储备营地 = 不动）  

### 次日：尾盘 / 盘后复核

1. 只对昨日名单里的票看 **BUY / WATCH / SKIP**  
2. **只有 BUY（可执行）才买**；WATCH、高位观察、涨停观察 = 不买  
3. 未 `confirmed`、无支撑、破支撑、冲高回落、禁新开水温 = 不买  

### 持仓

- 跟 **尾盘持仓段** + **Step4 OMS**  
- 优先级：`EXIT/TRIM > HOLD > PROBE/ATTACK`  
- 非主线满 **5 日**优先时间止盈；灾难地板约 **-12%**（不是日常洗盘线）  

---

## 5. 下单检查清单（缺一不可）

- [ ] 水温允许新开（不是 RISK_ON / 弱市）  
- [ ] 来自主线书（或明确轻仓的结构票）  
- [ ] Step3 为起跳板（若有研报）  
- [ ] 信号 **confirmed**  
- [ ] 尾盘决策 = **BUY（可执行）**  
- [ ] 未超过防追高限价 / 未高开失控  

---

## 6. 报告上哪里看规则

| 报告 | 纪律位置 |
|------|----------|
| 日漏斗飞书卡 | 顶部「今日执行纪律」+ 候选清单说明 |
| Step3 研报输入 | 宏观水温前的「执行纪律」 |
| 尾盘 / 盘后复核 | 统计后、买卖列表前 |
| Step4 OMS 工单 | 市场视图后 |
| 持仓上下文 | 「时间管理：TIME_EXIT / HOLD …」 |

---

## 7. 常见错误

1. 日漏斗出票就开盘追 → 买早  
2. 尾盘 WATCH 当 BUY → 未确认硬上  
3. RISK_ON 仍新开 → 与闸门对着干  
4. 用 -7% 当日常止损砍主升 → 被洗盘打掉  
5. 把观察池 / 旁路当主仓 → 负期望堆仓
